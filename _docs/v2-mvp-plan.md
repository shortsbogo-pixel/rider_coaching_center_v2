# Ver.2 MVP 개발 순서 제안

## MVP 목표

Ver.2 MVP는 매주 원천 엑셀 1개를 업로드해 관리자에게 주간 운영/검수/코칭 화면을 제공하고, 라이더에게는 본인 현황과 코칭 메시지만 보여주는 모바일 우선 웹앱으로 정의한다.

MVP에서 하지 않을 일:

- 정산 최종 확정 기능
- 교육/출석/복잡한 미션 관리
- AI 운영 브리핑, 메시지 발송 큐 전체
- 모든 시트의 정산/보험/차감 상세 반영
- 지도 좌표 자동 보정의 완성형 구현

## 1단계. IA와 데이터 계약 확정

완료 기준:

- 관리자 메뉴: 대시보드, 업로드, 검수, 코칭, 더보기
- 라이더 메뉴: 내현황, 내오더, 내지도, 내코칭, MY
- `weekly_uploads`, `orders`, `rider_week_metrics`, `coaching_messages`, `users`의 필드 확정
- 같은 주차 재업로드 정책 확정
- 라이더 식별/매칭 정책 확정

산출물:

- 라우트/화면 목록
- DB 스키마 초안
- 권한 매트릭스

## 2단계. 프로젝트 골격과 인증/권한 기반

완료 기준:

- admin/rider role 분기
- pending/active/inactive 계정 상태 처리
- 라이더는 서버 API에서도 본인 `rider_id` 데이터만 조회
- 관리자만 업로드/검수/코칭 관리 접근

비고:

- Ver.1 mock 인증은 참고만 하고, Ver.2는 실제 배포 가능한 인증 구조로 시작하는 것이 좋다.

## 3단계. 엑셀 업로드/파싱 MVP

완료 기준:

- 허용 파일명만 업로드 가능
- 정산/최종 파일명 차단
- `오더별 상세 내역서` 시트 존재 확인
- 7행 헤더와 B:Y 컬럼 구조 파싱
- 파일명에서 `YYYY_MM-W` 주차 코드 추출
- 파싱 미리보기에서 총 행, 정상 행, 이슈 행, 필수 컬럼 상태 표시
- 저장 전 검수 상태가 명확히 표시

샘플 검증 기준:

- `동구바로_대전_동구중앙_2026_05-4.xlsx` 처리 가능
- `정산` 또는 `최종`이 포함된 파일 차단

## 4단계. 업로드 검수 화면

완료 기준:

- 필수 컬럼 누락
- 라이더명 누락/미매칭
- 중복 오더 후보
- 피크타임 오류/빈 값
- 배달타입 확인필요
- 시간/거리/금액 변환 오류
- 검수 결과를 주차별로 확인

MVP에서는 수정 기능을 크게 만들기보다, “어떤 행이 왜 확인 필요한지”를 먼저 정확히 보여주는 것이 우선이다.

## 5단계. 라이더 주간 지표 계산

완료 기준:

- 라이더별 완료건수
- 출근/활동일
- 요일별 완료건수
- 구간별 완료건수와 강점/보강 구간
- 배달타입별 완료건수와 멀티율
- Post_Lunch/Post_Dinner 참여율
- 배차 친화 점수
- 등급 또는 목표 힌트

주의:

- Ver.1 점수식은 좋은 출발점이지만, 원시 샘플에 없는 거절/무시율은 MVP 점수에서 제외하거나 기본값 처리해야 한다.

## 6단계. 관리자 대시보드

완료 기준:

- 최신 주차 KPI
- 라이더 랭킹
- 구간 분석
- 최근 업로드 요약
- 검수 이슈 요약
- 코칭 필요 라이더 진입 링크

목표는 “이번 주 상태를 30초 안에 파악”하는 것이다.

## 7단계. 관리자 코칭 메시지 관리

완료 기준:

- 주차 선택
- 라이더 선택/검색
- 자동 메시지 생성
- 관리자 수정 메시지 저장
- 라이더 노출 ON/OFF
- 자동 메시지로 되돌리기
- 관리자 내부 메모는 라이더 노출 메시지와 분리
- 라이더 화면 미리보기

이 단계까지 끝나면 Ver.1의 핵심 가치였던 코칭 관리 흐름이 Ver.2 MVP에 들어온다.

## 8단계. 라이더 화면 MVP

완료 기준:

- `내현황`: 완료건수, 활동일, 점수, 주요 힌트
- `내오더`: 본인 오더 목록과 기본 필터
- `내지도`: 좌표가 없으면 지도 확장 예정 상태 또는 텍스트 기반 지역 요약으로 시작
- `내코칭`: 관리자 노출 ON 메시지, 배차 친화 점수, 구간 참여율, 코칭 포인트, 실천 미션
- `MY`: 계정 상태, 문의번호, 비밀번호/공지 영역

중요 원칙:

- 라이더 화면에는 타 라이더 선택 UI를 두지 않는다.
- 관리자 내부 메모, 검수 상세, 업로드 오류 내부 정보는 라이더에게 노출하지 않는다.

## 9단계. 테스트와 운영 리허설

완료 기준:

- 샘플 원천 파일 업로드 성공
- 제외 파일 업로드 차단
- 같은 주차 재업로드 플로우 확인
- 관리자 전체 조회와 라이더 본인 조회 권한 테스트
- 라이더가 다른 rider_id API를 호출해도 차단
- 코칭 메시지 ON/OFF 표시 테스트
- 모바일 화면 폭에서 하단 탭과 카드가 깨지지 않음

## 권장 개발 순서 요약

1. 데이터 계약/권한 정책 확정
2. 프로젝트 골격 및 인증/role 분기
3. 엑셀 업로드/파싱
4. 검수 화면
5. 라이더 지표 계산
6. 관리자 대시보드
7. 관리자 코칭 메시지 관리
8. 라이더 화면
9. 테스트/운영 리허설

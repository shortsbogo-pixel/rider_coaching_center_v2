## 2. Ver.1 주요 기능

- 프론트엔드: React + Vite + TypeScript 기반의 관리자/라이더 웹 화면.
- 백엔드: Express API, JSON 파일 저장소, repository/service/route 계층.
- 인증: MVP 테스트용 mock 계정과 role 기반 라우팅. 실제 배포 전 서버 인증으로 교체 필요.
- 업로드: 브라우저 파일 선택, `오더별 상세 내역서` 시트 탐색, 파싱 미리보기, 저장.
- 검수: 필수 컬럼 누락, 라이더명 누락, 배달타입 확인필요, 숫자 변환 경고, 시간대 오류, 이상치 집계.
- 분석: 라이더별 완료건수, 요일/구간/배달타입 분포, 멀티율, 포스트구간 참여율, 배차 친화 점수, 등급, 위험도 계산.
- 코칭: 라이더별 자동 메시지 생성, 관리자 수정 메시지 저장/초기화, 관리자 내부 메모 분리.
- 라이더 화면: 본인 운행 인사이트, 배차 친화 점수, 등급/다음 등급 힌트, 코칭 메시지, 실천 미션.
- 관리자 화면: 대시보드, 업로드, 검수, 라이더 분석, 미션 추천, 코칭 메시지, 데이터 관리, AI/운영 브리핑 관련 확장 기능.

## 3. Ver.1에서 유지할 라이더 기능

Ver.2 라이더 화면은 Ver.1의 “내 운행 인사이트” 경험을 핵심으로 유지하되, 메뉴는 기획서 기준 `내현황/내오더/내지도/내코칭/MY`로 단순화한다.

| 기능 | Ver.1 근거 | Ver.2 유지 방향 |
| --- | --- | --- |
| 본인 데이터만 조회 | rider role은 본인 `riderId` 외 API 접근 차단 | 라이더 API와 화면 모두 서버 기준으로 본인 데이터만 반환 |
| 주차 기준 선택 | 업로드된 주차 중 최신 주차 기본 선택 | 라이더 화면도 최신 주차 기본, 필요 시 본인 주차만 선택 |
| 완료건수 KPI | `totalCompleted`, `weeklyCompleted` | `내현황` 첫 카드와 `내코칭` 점수 근거로 사용 |
| 출근/활동 패턴 | `weekdayCompleted`, active weekday count | 출근일/활동 요일 카드로 유지 |
| 구간별 참여율 | Breakfast, Lunch_Peak, Post_Lunch, Dinner_Peak, Post_Dinner | 아코디언 또는 막대 차트로 강점/보강 구간 표시 |
| 배달타입/멀티율 | 단건/멀티배달1~4, 확인필요 | 멀티율 KPI와 코칭 조건에 사용 |
| 배차 친화 점수 | 완료건수, 멀티율, 포스트구간, 일관성, 거절/무시율 기반 | MVP에서는 원시 엑셀에 있는 필드만으로 계산하고, 없는 지표는 명시적으로 제외/기본값 처리 |
| 등급/다음 등급 힌트 | S/A/B/C/관리대상 기준 | 라이더가 이해하기 쉬운 “다음 목표 N건” 형태로 표시 |
| 코칭 포인트 | 강점, 보강점, 추천 행동, 수익 팁, 주간 미션 | `내코칭` 핵심 콘텐츠로 유지 |
| 관리자 수정 메시지 우선 표시 | custom message가 있으면 자동 메시지보다 우선 | 관리자 노출 ON 상태인 메시지만 라이더에게 표시 |

## 4. Ver.1에서 유지할 관리자 코칭 메시지 관리 기능

Ver.1의 코칭 메시지 관리는 Ver.2 핵심 메뉴인 `코칭`으로 승계한다.

| 기능 | 유지 여부 | Ver.2 방향 |
| --- | --- | --- |
| 기준 주차 선택 | 유지 | 업로드 주차 목록에서 선택 |
| 라이더 선택 | 유지 | 관리자만 전체 라이더 검색/필터 가능 |
| 자동 메시지 생성 | 유지 | 완료건수, 출근일, 구간 참여율, 멀티율, 점수 기반으로 생성 |
| 관리자 수정 메시지 | 유지 | 자동 메시지를 복사해 수정 저장 |
| 자동 메시지로 되돌리기 | 유지 | 수정 메시지 비활성 또는 초기화 |
| 관리자 내부 메모 | 유지하되 분리 | 라이더에게 절대 노출하지 않는 운영 메모로 관리 |
| 라이더 화면 미리보기 | 유지/고도화 | 저장 전 실제 `내코칭` 표시 형태로 확인 |
| 라이더 노출 여부 | Ver.2에서 필수 보강 | `visible_to_rider` 또는 상태값으로 ON/OFF 관리 |
| 주차별 이력 | 유지 | 같은 라이더라도 주차별 메시지를 별도 저장 |
| 템플릿 관리 | MVP 후반 또는 2차 | 점수/구간/출근 패턴별 템플릿은 기본 구조만 준비 |

## 5. Ver.1에서 그대로 가져오면 안 되는 부분

- Ver.1은 JSON 파일 저장소와 mock 인증 중심이다. Ver.2는 신규 앱이므로 DB/Auth/RLS 또는 동등한 서버 권한 구조를 새로 설계해야 한다.
- Ver.1의 AI 코칭, 운영 브리핑, 메시지 큐, 배포 점검 패널은 기능 범위가 넓다. MVP에서는 자동 메시지와 관리자 수정/노출 관리까지만 먼저 구현하는 편이 안전하다.
- Ver.1 파서는 `완료건수` 컬럼을 권장하지만 실제 샘플의 핵심 시트에는 해당 컬럼이 없다. Ver.2는 오더 1행을 기본 1건으로 계산하는 규칙을 명시해야 한다.
- 정산 확정, 보험료/차감/지원금 상세 처리, 교육/미션/출석/복잡한 운영 리포트는 MVP 이후로 분리한다.

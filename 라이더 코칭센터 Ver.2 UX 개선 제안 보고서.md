# 라이더 코칭센터 Ver.2 UX 개선 제안 보고서

현재 구현된 라이더 코칭센터 Ver.2 모바일 웹앱은 관리자의 엑셀 업로드 기반 데이터 관리와 라이더의 자가 페이스 체크를 핵심 가치로 삼고 있습니다. 분석 결과, 사용자 경험을 한 단계 높이기 위한 주요 개선 방향을 다음과 같이 제안합니다.

---

## 1. 관리자 경험 (Admin Experience)

### 1.1 엑셀 업로드 및 검수 프로세스 최적화
- **실시간 업로드 진행률 표시**: 현재 `isParsing` 상태만 보여주는데, 대용량 파일 처리 시 진행률(%) 또는 처리 단계를 시각화하여 대기 불안감을 해소해야 합니다.
- **오류 행 바로가기**: 검수 이슈 발생 시 해당 행 번호만 보여주는 대신, 클릭 시 해당 데이터의 상세 내용을 팝업으로 보여주거나 필터링하여 관리자가 즉각적으로 문제를 판단할 수 있게 해야 합니다.
- **업로드 히스토리 강화**: 현재는 세션 로그만 남지만, 과거 업로드된 주차별 요약 데이터를 비교 차트로 제공하여 운영 트렌드를 한눈에 파악하도록 개선이 필요합니다.

### 1.2 코칭 메시지 관리 효율화
- **템플릿 시스템**: 라이더별로 매번 메시지를 작성하는 대신, 성과(상위 10%, 하단 20% 등)나 이슈 유형에 따른 자동 추천 메시지 템플릿 기능을 도입하여 관리 공수를 줄여야 합니다.
- **일괄 전송/적용**: 특정 그룹의 라이더들에게 동일한 공지나 코칭을 한 번에 적용하는 기능을 추가하여 효율성을 높여야 합니다.

---

## 2. 라이더 경험 (Rider Experience)

### 2.1 페이스 체크의 직관성 및 입력 편의성
- **슬라이더/버튼 기반 입력**: 현재 '오늘 완료 콜 수' 등을 키보드로 직접 입력하는데, 운행 중인 라이더의 상황을 고려하여 큰 버튼(+/-)이나 슬라이더를 통해 한 손으로 쉽게 입력할 수 있도록 개선해야 합니다.
- **시각적 목표 달성도**: 숫자로 된 '추가 필요 건수' 외에도 게이지 바나 원형 프로그레스 바를 사용하여 목표 달성까지의 거리를 시각적으로 더 강렬하게 전달해야 합니다.
- **음악/루틴 모드 개인화**: 라이더가 선호하는 음악 장르나 루틴(주간/야간)을 한 번 설정하면 다음 접속 시에도 유지되도록 로컬 스토리지를 넘어선 프로필 저장 기능이 필요합니다.

### 2.2 정보 접근성 강화
- **내 현황 대시보드 커스텀**: 라이더가 가장 중요하게 생각하는 지표(수익, 콜 수, 안전 지수 등)를 상단에 배치할 수 있는 개인화 기능을 제안합니다.
- **지도 기반 오더 분석**: 단순히 목록으로 보여주는 '내 오더'를 지도상에 히트맵으로 표시하여, 어느 지역에서 주로 배달을 수행했는지 시각적으로 분석해 주는 기능이 유용할 것입니다.

---

## 3. 공통 및 시스템 UX

### 3.1 모바일 최적화 (Mobile-First)
- **제스처 내비게이션**: 탭 클릭 외에도 화면 스와이프를 통해 메뉴 간 이동이 가능하도록 하여 앱과 같은 사용성을 제공해야 합니다.
- **다크 모드 지원**: 야간 운행이 많은 라이더의 시력 보호와 배터리 절약을 위해 시스템 설정에 따른 자동 다크 모드 전환 기능이 필수적입니다.

### 3.2 데이터 신뢰성 및 안내
- **오프라인 모드 대응**: 통신이 불안정한 지역에서도 페이스 체크 입력이 가능하도록 데이터를 로컬에 선저장하고 연결 시 동기화하는 전략이 필요합니다.
- **세션 만료 안내**: 현재 인메모리 저장 방식의 한계를 사용자에게 더 명확하고 친절하게 안내하여 데이터 유실에 대한 당혹감을 최소화해야 합니다.

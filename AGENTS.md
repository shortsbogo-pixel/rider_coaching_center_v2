# 라이더 코칭센터 Ver.2 - Codex 작업 지침

## 프로젝트 목표
라이더 코칭센터 Ver.2는 기존 Ver.1을 수정하는 것이 아니라 신규 제작한다.
원시 엑셀 데이터 양식은 Ver.1과 동일하게 사용한다.
관리자는 주차별 라이더 운행 현황과 코칭 메시지를 관리하고,
라이더는 본인의 현황, 오더, 지도, 코칭 메시지만 확인한다.

## 핵심 메뉴
관리자:
- 대시보드
- 업로드
- 검수
- 코칭
- 더보기

라이더:
- 내현황
- 내오더
- 내지도
- 내코칭
- MY

## 엑셀 업로드 규칙
사용 파일:
- 동구바로_대전_동구중앙_YYYY_MM-W.xlsx

제외 파일:
- 정산 최종.xlsx
- 파일명에 "정산" 또는 "최종"이 포함된 파일

기본 시트:
- 오더별 상세 내역서

중요:
- 코드에 F:\ 또는 D:\ 같은 로컬 절대경로를 고정하지 않는다.
- 업로드는 브라우저 파일 선택 방식으로 구현한다.
- 같은 주차 재업로드 시 기존 데이터 교체 여부를 확인한다.

## 필수 기능
- 로그인
- 회원가입
- 관리자 승인 대기
- 관리자/라이더 권한 분기
- 관리자 엑셀 업로드
- 업로드 검수 결과
- 라이더별 주간 인사이트
- 관리자 코칭 메시지 관리
- 라이더 내코칭 화면
- 라이더 본인 데이터만 조회
- 관리자 전체 데이터 조회

## UI 방향
- 모바일 우선
- 카드형 대시보드
- 하단 탭 메뉴
- 큰 숫자 KPI
- 실무형 관리자 툴 느낌
- 라이더 화면은 간단하고 명확하게

## 배포 방향
- 외부 휴대폰 사용자가 접속 가능한 모바일 웹앱으로 만든다.
- 로컬 PC 전용 프로그램으로 만들지 않는다.
- Vercel 또는 유사한 웹 배포 환경을 전제로 한다.
- 관리자와 라이더는 같은 URL로 접속하되 로그인 후 role에 따라 화면을 분기한다.

## 작업 원칙
- 먼저 분석하고 계획을 제시한 뒤 코드 수정한다.
- Ver.1의 좋은 라이더 인사이트 기능은 살린다.
- 불필요한 메뉴는 추가하지 않는다.
- 테스트 가능한 단위로 구현한다.
- 실제 개인정보나 민감정보는 커밋하지 않는다.

# Rider Coaching Center Ver.2 Beta Test Checklist

## 링크 운영 정책 참조

정산앱과 코칭센터의 양방향 링크 운영 기준은 `app/docs/settlement-coaching-link-policy.md`를 따른다.

- 코칭센터에서 정산앱으로 이동하는 링크는 단순 외부 이동이다.
- 정산앱에서 코칭센터로 이동하는 링크는 정산앱 관리자 모드의 링크 설정 기준으로 관리한다.
- 두 서비스는 데이터 연동, SSO, 권한 통합을 하지 않는다.
- 정산 계산/지급/확정은 정산앱 기준이고, 운행 분석/코칭/오늘의 페이스 체크는 코칭센터 기준이다.

## 베타 테스트 목적

쿠팡플러스 주차별 확정 데이터와 라이더 수기 입력을 기반으로 관리자 코칭 흐름과 라이더 본인 화면을 소규모로 확인한다.

이 앱은 실시간 관제, 현재 위치 확인, 실시간 콜 추적 도구가 아니다.

## 테스트 계정

| 역할 | 아이디 | 비밀번호 | 확인 범위 |
| --- | --- | --- | --- |
| 관리자 | admin | admin1234 | 업로드, 검수, 코칭, 운영 로그, 페이스 설정 |
| 라이더 | rider1 | rider1234 | 본인 데이터, 오늘의 페이스 체크, 내코칭 |
| 승인대기 | pending | pending1234 | 승인 전 접근 제한 |

운영 배포 전에는 테스트 계정 안내를 `showTestAccounts` 플래그로 숨긴다.

## 확인 순서

1. 정상 엑셀 업로드
2. 미리보기 확인
3. 이번 주차 데이터로 반영
4. 대시보드 숫자 변경 확인
5. 검수 이슈 확인
6. 코칭 메시지 확인
7. 라이더 계정에서 본인 데이터만 확인
8. 오늘의 페이스 체크 입력
9. 관리자 페이스 설정 변경
10. 운영 로그 확인
11. 잘못된 파일 선택 시 기존 데이터 유지
12. 라이더 MY와 관리자 더보기에서 정산앱 링크 안내 확인

## 빈 상태 확인

- 업로드 데이터가 없을 때 샘플 데이터 안내가 보이는지 확인한다.
- 라이더 본인 데이터가 없을 때 계정 매핑 안내가 보이는지 확인한다.
- 페이스 체크 기준 데이터가 부족할 때 완료 수 중심 안내가 보이는지 확인한다.
- 새로고침 후 업로드 데이터가 사라질 수 있다는 안내가 보이는지 확인한다.

## 모바일 디자인 점검

- 360px 폭에서 하단 5개 메뉴의 아이콘과 텍스트가 겹치지 않는지 확인한다.
- 마지막 카드가 하단 메뉴에 가려지지 않는지 확인한다.
- 관리자 대시보드, 업로드, 검수, 코칭, 더보기 카드 간격과 버튼 높이를 확인한다.
- 라이더 내현황에서 오늘의 페이스 체크, 다음 추천 행동, 오늘 입력, 추가 필요가 첫 화면에서 이해되는지 확인한다.
- 내오더, 내지도, 내코칭, MY 화면의 안내 문구가 카드 밖으로 넘치지 않는지 확인한다.
- 성공/주의/위험/정보 상태가 색상과 텍스트로 함께 구분되는지 확인한다.
- 라이더 MY의 정산 확인 카드와 관리자 더보기의 정산 관리 앱 카드가 360px 폭에서 잘리지 않는지 확인한다.

## 정산앱 링크 확인

- 정산앱 링크는 기존 정산앱으로 이동하는 단순 외부 링크이다.
- 코칭센터는 정산 금액 계산, 정산 엑셀 파싱, 정산 DB/API 연동을 하지 않는다.
- 라이더 MY의 `정산앱 열기` 버튼이 새 탭 또는 새 창으로 열리는지 확인한다.
- 관리자 더보기의 `정산 관리 앱 열기` 버튼이 새 탭 또는 새 창으로 열리는지 확인한다.
- 정산 계산, 지급, 확정, 인증과 권한은 기존 정산앱에서 처리한다.

## 베타 제외 항목

- DB 저장
- 외부 접속 정식 배포
- 정산 계산
- 정산 엑셀 파싱
- 정산앱 데이터 연동
- 실시간 콜 연동
- GPS/날씨
- 실제 음악 재생
- 문자 발송
- 타 플랫폼 확장

# Beta Feedback Form

## 베타 공개 안내문 참조

테스트 참여자에게 보낼 짧은 안내문은 `app/docs/beta-release-messages.md`를 기준으로 공유한다.

- 피드백 요청 시 화면 캡처, 계정 유형, 발생 메뉴, 기대 동작을 함께 요청한다.
- 실시간 앱이 아니며 최신 업로드 주차 기준 코칭 앱이라는 안내를 포함한다.
- 정산 상세와 지급 상태는 기존 정산앱에서 확인한다는 안내를 포함한다.

## 목적

라이더 코칭센터 Ver.2 1차 베타 테스트에서 나온 의견을 같은 기준으로 수집하고, 오류와 개선 요청을 분리해 후속 작업 우선순위를 정한다.

이번 피드백 수집은 기능 개발이 아니라 베타 운영 정리용이다. 실제 개인정보, 원천 엑셀 파일, 정산 원본 파일은 첨부하지 않는다.

## 베타 종료 후 정리 방식

1. 중복 피드백 묶기
   - 같은 화면, 같은 증상, 같은 원인은 하나의 대표 항목으로 묶는다.
   - 기기나 브라우저별 차이가 있으면 같은 항목 안에 조건을 남긴다.

2. 긴급/높음 우선 처리
   - 로그인 불가, 화면 진입 불가, 업로드 불가, 권한 노출 문제를 먼저 분리한다.
   - 주요 수치 오류와 정산앱 링크 실패도 우선 확인한다.

3. 기능 추가 요청 분리
   - 현재 베타 범위를 넘어서는 요청은 2차 개발 후보로 분리한다.
   - 문자 발송, GPS, 실시간 콜, 실제 음악 재생, 정산 계산은 이번 베타 범위가 아니다.

4. DB/Supabase 관련 요청 분리
   - 새로고침 후 데이터 유지, 계정별 저장, 운영 로그 보존 요청은 DB/Supabase 도입 판단 항목으로 분리한다.
   - 실제 DB 연결은 운영 전 저장 정책, 권한 정책, 개인정보 범위가 결정된 뒤 진행한다.

5. 정산앱 관련 요청 구분
   - 지급 상태, 정산 확정, 상세 원장, 정산 계산 관련 요청은 기존 정산앱 개선 요청으로 분리한다.
   - 코칭센터는 정산앱으로 이동하는 링크만 제공한다.

## 개인정보 주의

- 실제 전화번호, 계좌번호, 상세 주소, 주민등록번호는 피드백 문서에 적지 않는다.
- 원천 엑셀 파일, CSV, 정산 최종 파일을 피드백에 첨부하지 않는다.
- 캡처가 필요하면 라이더 개인정보나 정산 민감정보가 보이지 않게 가린다.
- 라이더 이름이 꼭 필요하지 않으면 `rider_id` 또는 테스트 계정 유형으로 기록한다.

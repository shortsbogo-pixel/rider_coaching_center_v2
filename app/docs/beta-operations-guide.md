# Rider Coaching Center Ver.2 Beta Operations Guide

## 실제 화면 점검 리포트 참조

베타 공개 직전 실제 Vercel 배포 URL 기준 화면 점검 결과는 `app/docs/beta-release-screen-check-report.md`에 기록한다.

- 기본 접속, 테스트 계정, 관리자/라이더/pending 화면을 확인한다.
- 코칭센터와 정산앱 양방향 링크를 확인한다.
- 모바일 360px 기준 화면 깨짐과 하단 메뉴 겹침 여부를 확인한다.

## 피드백 우선순위 문서 참조

1차 베타 중 들어오는 피드백의 반영 순서는 `app/docs/beta-feedback-priority-plan.md`를 기준으로 정한다.

- 긴급/높음 항목은 베타 진행 중 우선 확인한다.
- 보통/낮음 항목은 UX 개선 목록으로 분리한다.
- 기능 추가 요청과 DB/Supabase 요청은 2차 개발 또는 별도 의사결정 항목으로 분리한다.

## 베타 공개 안내문 참조

관리자/라이더/승인대기 계정에 실제로 공유할 짧은 카카오톡/문자 안내문은 `app/docs/beta-release-messages.md`를 사용한다.

- 관리자용 안내문은 업로드, 검수, 반영, 코칭, 운영 로그, 정산앱 링크 확인을 요청한다.
- 라이더용 안내문은 내현황, 오늘의 페이스 체크, 내오더, 내지도, 내코칭, MY 확인을 요청한다.
- 승인대기 안내문은 데이터 화면 접근 불가 확인을 요청한다.
- 내부 운영자용 안내문은 테스트 순서와 피드백 분류 기준을 포함한다.

## 최종 공개 전 체크리스트 참조

1차 베타 참여자에게 라이더 코칭센터 URL을 공유하기 전에는 `app/docs/beta-release-final-checklist.md`를 기준으로 최종 운영 점검을 진행한다.

- 배포 접속, 모바일 360px, 테스트 계정 정책을 확인한다.
- 관리자 업로드/검수/코칭/운영 로그와 라이더 주요 화면을 확인한다.
- 정산앱 양방향 링크와 데이터 비연동 안내를 확인한다.
- 공개 가능, 일부 보완 후 공개, 공개 보류 중 하나로 판단 기록을 남긴다.

## 목적

라이더 코칭센터 Ver.2 1차 베타 테스트를 관리자와 라이더가 같은 기준으로 확인할 수 있도록 운영 안내와 테스트 순서를 정리한다.

이번 베타의 목적은 쿠팡플러스 주차별 확정 엑셀 데이터 기반 코칭 흐름과 라이더 본인 화면을 확인하는 것이다. 실시간 관제, 현재 위치 확인, 실시간 콜 추적 앱이 아니다.

## 접속 주소

| 구분 | 주소 | 용도 |
| --- | --- | --- |
| 라이더 코칭센터 | https://rider-coaching-center-v2.vercel.app | 관리자/라이더 베타 테스트 |
| 정산앱 | https://ridersettle-fmyozrbw.manus.space/ | 정산 상세, 지급 상태, 상세 원장 확인 |

정산앱은 별도 시스템이다. 코칭센터는 정산앱으로 이동하는 링크만 제공하며 정산 계산, 지급, 확정, 정산 엑셀 파싱을 하지 않는다.

## 테스트 계정

| 역할 | 아이디 | 비밀번호 | 확인 범위 |
| --- | --- | --- | --- |
| 관리자 | `admin` | `admin1234` | 업로드, 검수, 코칭, 운영 로그, 페이스 설정, 정산앱 링크 |
| 라이더 | `rider1` | `rider1234` | 본인 데이터, 오늘의 페이스 체크, 내오더, 내지도, 내코칭, MY |
| 승인대기 | `pending` | `pending1234` | 승인 전 접근 제한 화면 |

운영 공개 전에는 테스트 계정 안내를 숨기고 개인 계정 체계로 전환해야 한다.

## 관리자용 베타 안내

### 접속

1. https://rider-coaching-center-v2.vercel.app 접속
2. `admin / admin1234`로 로그인
3. 하단 메뉴가 `대시보드`, `업로드`, `검수`, `코칭`, `더보기`로 보이는지 확인

### 대시보드 체크 포인트

- 샘플 데이터 또는 업로드 데이터 기준 문구가 보이는지 확인
- 완료 콜, 운영 라이더, 검수 이슈, 노출 메시지 숫자가 보이는지 확인
- 최근 업로드 요약과 마지막 반영 시간이 이해되는지 확인
- 업로드 전에는 샘플 데이터 안내가 명확히 보이는지 확인

### 업로드 체크 포인트

- 정상 엑셀 파일 선택 영역이 보이는지 확인
- 파일명 규칙: `동구바로_대전_동구중앙_YYYY_MM-W.xlsx`
- `정산` 또는 `최종`이 포함된 파일은 차단되는지 확인
- 파일 선택 후 바로 앱 전체에 반영되지 않고 미리보기만 보이는지 확인
- 총행수, 유효 오더 후보, 확인 필요 행, 이슈 건수, 라이더별 집계 미리보기가 보이는지 확인
- `이번 주차 데이터로 반영`을 눌렀을 때만 대시보드/검수/코칭/라이더 화면이 바뀌는지 확인
- `미리보기 취소` 시 기존 반영 데이터가 유지되는지 확인

### 검수 체크 포인트

- 총행수, 유효 오더 후보, 확인 필요 행, 이슈 건수가 구분되어 보이는지 확인
- 배달타입 누락, 피크타임 누락, 이름 누락, 주문번호 누락, 시간값 누락 이슈가 보이는지 확인
- 이슈 건수는 중복 포함 기준임을 이해할 수 있는지 확인
- 잘못된 파일 선택 후 기존 정상 데이터가 유지되는지 확인

### 코칭 체크 포인트

- 최신 반영 주차 기준 라이더 목록이 보이는지 확인
- 라이더명, 완료건수, 배차 친화 점수, 구간별 참여율, 멀티비율이 보이는지 확인
- 자동 메시지, 관리자 수정 메시지, 노출 ON/OFF, 내부 메모가 보이는지 확인
- 라이더 화면 미리보기에서 노출 메시지가 어떻게 보이는지 확인

### 더보기 체크 포인트

- 베타 테스트 안내와 베타 체크리스트가 보이는지 확인
- 페이스 체크 설정을 확인하고, 변경 후 라이더 화면 기준이 바뀌는지 확인
- 운영 로그에 미리보기, 반영, 취소, 차단, 실패, 설정 변경 이력이 남는지 확인
- 정산 관리 앱 카드가 보이는지 확인
- `정산 관리 앱 열기` 버튼이 새 탭 또는 새 창으로 정산앱을 여는지 확인

## 라이더용 베타 안내

### 접속

1. https://rider-coaching-center-v2.vercel.app 접속
2. `rider1 / rider1234`로 로그인
3. 하단 메뉴가 `내현황`, `내오더`, `내지도`, `내코칭`, `MY`로 보이는지 확인

### 중요한 안내

라이더 화면은 실시간 배달 화면이 아니다.

- 최신 업로드 주차의 과거 운행 이력을 보여준다.
- 관리자가 업로드하고 반영한 엑셀 데이터를 기준으로 표시된다.
- 현재 위치, 실시간 콜 진행, 배달 관제 기능은 없다.
- 라이더는 본인 `rider_id` 기준 데이터만 볼 수 있다.

### 내현황 체크 포인트

- 샘플 데이터 또는 업로드 데이터 기준 안내가 보이는지 확인
- 오늘의 페이스 체크가 첫 화면에서 이해되는지 확인
- 지난주 쿠팡플러스 확정 데이터 기준 문구가 보이는지 확인
- 오늘 완료 콜 수, 이번 주 목표, 목표까지 필요한 추가 콜 수가 보이는지 확인
- 다음 추천 행동 문구가 짧고 명확한지 확인

### 오늘의 페이스 체크 체크 포인트

- 오늘 완료 콜 수 입력 가능 여부 확인
- 오늘 운행 시작 시간 입력 가능 여부 확인
- 컨디션: 좋음, 보통, 피곤, 위험 선택 가능 여부 확인
- 식사 여부: 완료, 아직, 건너뜀 선택 가능 여부 확인
- 수면 시간 입력 가능 여부 확인
- 휴식 여부: 충분, 부족, 없음 선택 가능 여부 확인
- 주간형/야간형 루틴 선택에 따라 문구가 바뀌는지 확인
- 음악 모드는 선택 UI만 있고 실제 재생은 하지 않는지 확인
- 이 기기 기준 임시 저장 안내가 이해되는지 확인

### 내오더 체크 포인트

- 최신 업로드 주차의 운행 내역이라는 안내가 보이는지 확인
- 관리자가 업로드한 엑셀 데이터 기준이라는 안내가 보이는지 확인
- 타 라이더 선택 UI가 없는지 확인
- 본인 오더만 보이는지 확인

### 내지도 체크 포인트

- 실시간 위치 지도가 아니라 최신 업로드 주차의 픽업/배달 지역 기반 미리보기임을 확인
- 현재 위치 또는 실시간 관제로 오해될 표현이 없는지 확인

### 내코칭 체크 포인트

- 배차 친화 점수, 구간별 참여율, 출근/활동 패턴, 배달타입/멀티 정보가 보이는지 확인
- 코칭 포인트와 실천 미션이 이해되는지 확인
- 관리자가 노출한 코칭 메시지만 보이는지 확인

### MY 체크 포인트

- 계정 상태가 보이는지 확인
- 베타 테스트 안내가 보이는지 확인
- 정산 확인 카드가 보이는지 확인
- `정산앱 열기` 버튼이 새 탭 또는 새 창으로 정산앱을 여는지 확인
- 정산 상세와 지급 상태는 기존 정산앱에서 확인한다는 문구가 보이는지 확인

## 베타 테스트 진행 순서

### 1. 관리자 테스트 순서

1. 코칭센터 접속
2. `admin / admin1234` 로그인
3. 대시보드 샘플 데이터 안내 확인
4. 업로드 화면에서 정상 엑셀 파일 선택
5. 미리보기 숫자와 라이더별 집계 확인
6. `이번 주차 데이터로 반영` 클릭
7. 대시보드 숫자 변경 확인
8. 검수 화면 숫자와 이슈 유형 확인
9. 코칭 화면 자동 메시지와 노출 ON/OFF 확인
10. 더보기 화면 운영 로그 확인
11. 페이스 체크 설정 확인
12. 정산 관리 앱 링크 확인

### 2. 라이더 테스트 순서

1. 로그아웃 후 `rider1 / rider1234` 로그인
2. 내현황에서 데이터 기준 안내 확인
3. 오늘의 페이스 체크 입력
4. 목표까지 필요한 추가 콜 수와 다음 추천 행동 확인
5. 내오더에서 최신 업로드 주차 기준 과거 운행 내역 확인
6. 내지도에서 지역 기반 미리보기 확인
7. 내코칭에서 코칭 메시지 확인
8. MY에서 정산앱 링크 확인
9. 타 라이더 선택 UI가 없는지 확인

### 3. 승인대기 계정 테스트 순서

1. 로그아웃 후 `pending / pending1234` 로그인
2. 승인 대기 화면만 보이는지 확인
3. 관리자/라이더 데이터 화면에 접근할 수 없는지 확인

### 4. 정산앱 왕복 링크 테스트 순서

1. 관리자 더보기에서 `정산 관리 앱 열기` 클릭
2. 새 탭 또는 새 창으로 https://ridersettle-fmyozrbw.manus.space/ 열림 확인
3. 코칭센터 탭으로 돌아오기
4. 라이더 MY에서 `정산앱 열기` 클릭
5. 새 탭 또는 새 창으로 같은 정산앱이 열리는지 확인
6. 코칭센터가 정산 계산이나 지급 상태를 직접 처리하는 것처럼 보이지 않는지 확인

## 베타 제외 기능

이번 베타에서 제외한다.

- DB 저장
- Supabase 연결
- 외부 API
- 실시간 콜 연동
- 현재 위치/GPS
- 문자 발송
- 실제 음악 재생
- 정산 계산
- 정산 엑셀 파싱
- 정산앱 데이터 연동
- 타 플랫폼 확장

## 운영 전 전환 항목

- 테스트 계정 숨김: `showTestAccounts`를 운영 공개 전 숨김 처리
- 개인 계정 전환: 관리자/라이더별 실제 계정과 권한 체계 결정
- DB 도입 판단: Supabase/PostgreSQL 저장 도입 여부 결정
- 원천 엑셀 미저장 원칙: 원천 엑셀 파일, 브라우저 `File`, 업로드 바이너리는 저장하지 않음
- 개인정보 최소화: `rider_id`, 표시 이름, 주차별 집계 중심으로 저장 범위 제한
- 재업로드 정책: 같은 주차 업로드 시 교체, 이력 보존, 롤백 기준 결정
- 접근 범위: 베타 URL 공유 대상과 기간 제한
- xlsx 보안 판단: 운영 전 파일 크기 제한, 업로드 권한, 취약점 대응 결정

# Manus Handoff

## 핵심 요약

라이더 코칭센터 Ver.2는 쿠팡플러스 전용 모바일 웹앱 베타이다.

Manus 임시 배포 또는 외부 휴대폰 확인의 목적은 정식 운영이 아니라 화면, 업로드, 검수, 코칭, 라이더 본인 화면 흐름을 소규모로 검증하는 것이다.

- 실시간 관제 앱이 아니다.
- 현재 위치 확인, 실시간 콜 추적, 배달 진행 관제 기능은 없다.
- 주차별 확정 엑셀 데이터를 관리자가 브라우저에서 선택해 파싱한다.
- 원천 엑셀 파일은 코드, 저장소, 배포물에 포함하지 않는다.
- 현재 저장은 in-memory/session 기준이며 새로고침 또는 재배포 후 다시 업로드가 필요할 수 있다.
- 베타 목적은 쿠팡플러스 주차별 확정 데이터 기반 코칭 화면과 라이더 수기 입력 흐름 검증이다.
- 정산 확인은 기존 정산웹으로 이동하는 단순 링크이며, 코칭센터와 정산웹은 데이터를 직접 연동하지 않는다.

## 프로젝트 목적

관리자는 쿠팡플러스 주차별 확정 운행 데이터를 업로드하고, 검수 결과와 라이더별 코칭 메시지를 확인한다.

라이더는 로그인 후 본인 `rider_id` 기준의 최신 반영 주차 데이터만 확인하며, 오늘의 페이스 체크에서 현재 주차 목표와 컨디션을 직접 입력한다.

## 현재 기능 범위

- mock 로그인과 역할 분기
- 관리자 대시보드
- 관리자 엑셀 업로드, 파싱, 검수 미리보기
- 관리자의 이번 주차 데이터 반영
- 관리자 검수 화면
- 관리자 코칭 메시지 관리
- 관리자 운영 로그
- 관리자 페이스 체크 설정
- 관리자 정산 관리 웹 링크 안내
- 라이더 내현황, 내오더, 내지도, 내코칭, MY
- 라이더 오늘의 페이스 체크
- 라이더 정산 확인 링크 안내
- 승인대기 계정 접근 제한
- 베타 안내와 테스트 계정 안내

## 실행 명령

`app` 폴더에서 실행한다.

```bash
npm install
npm run dev
```

로컬 개발 서버 기본 URL은 `http://localhost:3000`이다. Manus 임시 배포에서는 Manus가 제공하는 외부 URL로 확인한다.

## 빌드 명령

```bash
npm run build
```

이 앱은 Next.js App Router 기반이다. Node.js 기반 배포 환경에서는 production build 후 `npm run start`로 실행할 수 있다.

## 테스트 명령

```bash
npm test
npm run lint
npm run build
```

배포 인계 전에는 위 세 명령이 모두 통과해야 한다.

## 베타 테스트 계정

테스트 계정 안내는 `src/lib/beta.ts`의 `betaMode`와 `showTestAccounts` 플래그 기준으로 노출된다.

| 역할 | 아이디 | 비밀번호 | 확인 범위 |
| --- | --- | --- | --- |
| 관리자 | `admin` | `admin1234` | 업로드, 검수, 코칭, 운영 로그, 페이스 설정 |
| 라이더 | `rider1` | `rider1234` | 본인 주차 데이터, 오늘의 페이스 체크, 내코칭 |
| 승인대기 | `pending` | `pending1234` | 승인 전 접근 제한 화면 |

운영 배포 전에는 `betaMode` 또는 `showTestAccounts`를 `false`로 바꿔 테스트 계정 안내를 숨긴다.

## 베타 테스트 절차

1. 로그인 화면에서 베타 안내와 테스트 계정 안내를 확인한다.
2. `admin / admin1234`로 로그인한다.
3. 관리자 업로드 화면에서 정상 엑셀 파일을 선택한다.
4. 파일명, 시트, 총행수, 유효 오더 후보, 확인 필요 행, 이슈 건수, 라이더별 집계를 미리보기에서 확인한다.
5. 미리보기 상태에서 대시보드 숫자가 아직 바뀌지 않는지 확인한다.
6. `이번 주차 데이터로 반영`을 누른다.
7. 관리자 대시보드, 검수, 코칭 화면이 반영 데이터 기준으로 바뀌는지 확인한다.
8. 관리자 더보기에서 운영 로그가 최신순으로 남는지 확인한다.
9. 로그아웃 후 `rider1 / rider1234`로 로그인한다.
10. 라이더 내현황, 내오더, 내지도, 내코칭에서 본인 데이터만 보이는지 확인한다.
11. 라이더 오늘의 페이스 체크에 오늘 완료 콜 수, 목표, 컨디션, 식사, 수면, 휴식을 입력한다.
12. 추천 행동과 목표까지 필요한 추가 콜 수가 입력값에 맞게 바뀌는지 확인한다.
13. 라이더 MY에서 정산 확인 카드가 보이고, 정산웹 주소 미설정 안내가 보이는지 확인한다.
14. 로그아웃 후 `pending / pending1234`로 로그인해 승인대기 화면만 보이는지 확인한다.
15. 정산/최종 파일 또는 시트가 없는 파일을 선택했을 때 기존 정상 반영 데이터가 유지되는지 확인한다.

## 외부 휴대폰 확인 체크리스트

- [ ] 로그인 화면이 모바일 폭에서 깨지지 않는다.
- [ ] 베타 안내가 짧고 명확하게 보인다.
- [ ] 관리자 업로드 화면에서 파일 선택 영역이 보인다.
- [ ] 정상 엑셀 업로드 후 미리보기와 반영 버튼이 보인다.
- [ ] 반영 후 관리자 대시보드 숫자가 바뀐다.
- [ ] 검수 화면에서 총행수, 유효 오더 후보, 확인 필요 행, 이슈 건수가 보인다.
- [ ] 코칭 화면에서 라이더별 메시지와 미리보기가 보인다.
- [ ] 운영 로그 화면에서 반영/취소/실패/차단 로그가 보인다.
- [ ] 관리자 더보기에서 정산 관리 웹 카드와 주소 미설정 안내가 보인다.
- [ ] 라이더 내현황에서 오늘의 페이스 체크가 먼저 이해된다.
- [ ] 라이더 내오더, 내지도, 내코칭이 최신 업로드 주차 기준임을 안내한다.
- [ ] 라이더 MY에서 정산 확인 카드와 주소 미설정 안내가 보인다.
- [ ] 새로고침 후 다시 업로드가 필요할 수 있다는 안내가 보인다.
- [ ] 하단 5개 메뉴가 360px 폭에서도 겹치지 않는다.

## 운영 전 주의사항

- 테스트 계정 안내를 숨길지 결정한다.
- 실제 인증 방식과 접근 제한 정책을 결정한다.
- 원천 데이터가 아닌 정규화 데이터 저장 방식을 결정한다.
- 같은 주차 재업로드 시 교체 또는 이력 보존 정책을 결정한다.
- `xlsx` 패키지 취약점 3건에 대해 운영 전 위험 수용, 완화, 대체 라이브러리 검토 여부를 결정한다.
- DB/Supabase 도입 여부와 도입 시점을 결정한다.
- 라이더 개인정보와 주차별 운행 데이터의 노출 범위를 점검한다.
- 운영 로그에 원천 파일 바이너리, 브라우저 `File` 객체, 불필요한 개인정보가 남지 않는지 확인한다.
- 외부 URL 공유 전 접근 가능한 대상과 테스트 기간을 제한한다.

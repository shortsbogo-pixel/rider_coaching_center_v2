# 정산앱 / 코칭센터 링크 운영 정책

## 목적

라이더 정산앱과 라이더 코칭센터 Ver.2의 양방향 링크 연결 상태와 운영 절차를 정리한다.

이번 문서는 운영 문서화 단계다. 기능 코드, 정산앱 코드, DB/Supabase 연결, 외부 API 연결, Vercel/GitHub 배포 설정은 변경하지 않는다.

## 현재 주소

| 서비스 | URL | 역할 |
| --- | --- | --- |
| 라이더 코칭센터 Ver.2 | https://rider-coaching-center-v2.vercel.app | 운행 분석, 코칭, 오늘의 페이스 체크 |
| 라이더 정산앱 | https://ridersettle-fmyozrbw.manus.space/ | 정산 상세, 지급 상태, 확정/상세 원장 |

## 양방향 링크 구조

두 서비스는 직접 데이터를 연동하지 않고 외부 링크 이동 방식으로만 연결한다.

| 방향 | 방식 | 책임 영역 |
| --- | --- | --- |
| 코칭센터 → 정산앱 | 코칭센터 화면의 외부 링크 버튼 | 정산 상세와 지급 상태를 정산앱에서 확인하도록 이동 |
| 정산앱 → 코칭센터 | 정산앱 관리자 설정으로 노출되는 외부 링크 버튼 | 운행 분석과 코칭 화면을 코칭센터에서 확인하도록 이동 |

역할 구분:

- 정산 계산, 지급, 확정, 상세 원장은 정산앱 기준이다.
- 운행 분석, 주차별 코칭, 오늘의 페이스 체크는 코칭센터 기준이다.
- 코칭센터는 정산앱을 대체하지 않는다.
- 정산앱은 코칭센터의 운행 분석/코칭 기능을 대체하지 않는다.
- 두 서비스의 링크는 데이터 연동, 권한 통합, 자동 로그인을 의미하지 않는다.

## 코칭센터에서 정산앱으로 이동

코칭센터에는 정산앱으로 이동하는 외부 링크가 있다.

노출 위치:

- 라이더 MY 화면
- 관리자 더보기 화면

설정 기준:

- 정산앱 URL은 `src/lib/settlement-link.ts`의 `SETTLEMENT_WEB_URL` 상수 한 곳에서 관리한다.
- 현재 값은 `https://ridersettle-fmyozrbw.manus.space/`이다.
- URL이 비어 있으면 버튼을 비활성화하고 주소 설정 전 안내를 표시하는 구조를 유지한다.

링크 동작:

- 새 탭 또는 새 창으로 열린다.
- `target="_blank"`를 사용한다.
- `rel="noopener noreferrer"`를 유지한다.
- 정산앱 인증과 권한은 정산앱에서 처리한다.

문구 원칙:

- 라이더 MY 화면은 “정산 상세와 지급 상태는 기존 정산앱에서 확인합니다.”를 유지한다.
- 관리자 더보기 화면은 “정산 확정, 지급 상태, 상세 원장은 기존 정산앱에서 처리합니다.”를 유지한다.
- 코칭센터는 운행 데이터와 코칭 확인용이라는 설명을 유지한다.

## 정산앱에서 코칭센터로 이동

정산앱에는 코칭센터로 이동하는 링크 설정이 별도로 존재한다.

정산앱 관리자 모드 설정 항목:

- “라이더 코칭센터 링크 설정”
- 링크 노출 ON/OFF
- 코칭센터 URL 입력
- 버튼 문구 입력
- 안내 문구 입력

운영 기준:

- 코칭센터 URL이 변경되면 정산앱 관리자 모드에서 URL을 수정할 수 있다.
- 링크 노출 ON/OFF는 정산앱 운영자가 관리한다.
- 버튼 문구와 안내 문구는 정산앱 화면에서 코칭센터가 정산앱을 대체하는 것처럼 보이지 않게 작성한다.
- 정산앱에서 코칭센터로 이동하더라도 정산앱 계정과 코칭센터 계정이 자동으로 통합되지는 않는다.

권장 버튼/안내 문구:

| 항목 | 권장 문구 |
| --- | --- |
| 버튼 문구 | 라이더 코칭센터 열기 |
| 안내 문구 | 운행 분석과 코칭 확인은 라이더 코칭센터에서 확인합니다. |

## 운영자가 코칭센터 URL을 변경할 때 절차

코칭센터 배포 URL이 바뀌는 경우 아래 순서로 정산앱 설정을 갱신한다.

1. 새 코칭센터 URL을 확인한다.
2. 정산앱 관리자 모드에 접속한다.
3. “라이더 코칭센터 링크 설정” 영역으로 이동한다.
4. 라이더 코칭센터 URL을 새 주소로 입력한다.
5. 링크 노출 ON 상태를 확인한다.
6. 버튼 문구와 안내 문구를 확인한다.
7. 저장한다.
8. 정산앱 라이더 화면에서 버튼 클릭 테스트를 한다.
9. 새 탭 또는 새 창으로 코칭센터가 열리는지 확인한다.
10. 코칭센터에서 정산앱 이동 테스트를 한다.

## 운영자가 정산앱 URL을 변경할 때 절차

정산앱 배포 URL이 바뀌는 경우 코칭센터 코드의 설정 상수를 변경한 뒤 재배포해야 한다.

1. 새 정산앱 URL을 확인한다.
2. 코칭센터의 `src/lib/settlement-link.ts`에서 `SETTLEMENT_WEB_URL`을 새 주소로 변경한다.
3. `npm test`, `npm run lint`, `npm run build`를 실행한다.
4. Vercel 또는 운영 배포 환경에 코칭센터를 재배포한다.
5. 라이더 MY 화면에서 `정산앱 열기` 버튼을 테스트한다.
6. 관리자 더보기 화면에서 `정산 관리 앱 열기` 버튼을 테스트한다.
7. 새 탭 또는 새 창으로 정산앱이 열리는지 확인한다.

주의:

- 실제 URL은 한 파일에서만 관리한다.
- 여러 화면에 URL을 직접 하드코딩하지 않는다.
- 정산앱 URL 변경은 기능 추가가 아니라 링크 설정 변경으로 취급한다.

## 베타 테스트 확인 항목

| 확인 항목 | 기준 |
| --- | --- |
| 정산앱에서 코칭센터 열림 | 정산앱 링크 버튼 클릭 시 코칭센터 URL이 열린다. |
| 코칭센터에서 정산앱 열림 | 라이더 MY/관리자 더보기 버튼 클릭 시 정산앱 URL이 열린다. |
| 새 탭 열림 | 기존 화면을 잃지 않고 새 탭 또는 새 창으로 열린다. |
| 모바일 탭 전환 | 뒤로가기/탭 전환 시 사용자가 길을 잃지 않는다. |
| 역할 오해 없음 | 정산앱이 코칭센터를 대체하는 것처럼 보이지 않는다. |
| 정산 계산 오해 없음 | 코칭센터가 정산 금액을 계산하는 것처럼 보이지 않는다. |
| 권한 통합 오해 없음 | 두 서비스가 자동 로그인/SSO처럼 보이지 않는다. |

모바일 확인 포인트:

- 360px 폭에서 링크 카드가 잘리지 않는다.
- 버튼 문구가 줄바꿈되어도 의미가 깨지지 않는다.
- 링크 이동 후 기존 탭으로 돌아오는 흐름이 혼란스럽지 않다.

## 운영 주의사항

- Vercel URL이 바뀌면 정산앱 관리자 설정도 변경해야 한다.
- 정산앱 URL이 바뀌면 코칭센터의 `SETTLEMENT_WEB_URL` 수정 후 재배포해야 한다.
- 정산앱 링크는 정산 데이터 연동이 아니다.
- 정산앱 링크는 SSO나 자동 로그인이 아니다.
- 정산앱 링크는 권한 통합이 아니다.
- 정산앱과 코칭센터는 각각 별도 책임 영역을 유지한다.
- 정산앱 인증/권한/지급 상태/확정 처리는 정산앱에서 관리한다.
- 코칭센터 로그인/권한/코칭 데이터 노출은 코칭센터에서 관리한다.
- 정산/최종 파일 업로드 차단 정책은 코칭센터에서 계속 유지한다.
- 원천 엑셀 파일과 정산 원본 파일은 코칭센터 배포물에 포함하지 않는다.

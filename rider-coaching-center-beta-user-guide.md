# 라이더 코칭센터 Ver.2 1차 베타 사용설명서

## 1. 문서 목적

이 문서는 라이더 코칭센터 Ver.2 1차 베타 테스트에 참여하는 관리자와 라이더가 각 화면의 기능을 이해하고, 정해진 방식으로 테스트할 수 있도록 만든 사용설명서입니다.

관리자는 엑셀 업로드, 데이터 검수, 코칭 메시지 관리, 운영 로그 확인을 담당합니다.

라이더는 본인 기준의 운행 현황, 오더 이력, 지도 미리보기, 코칭 메시지, 오늘의 페이스 체크를 확인합니다.

## 2. 서비스 기본 개념

라이더 코칭센터 Ver.2는 쿠팡플러스 주차별 확정 운행 데이터를 기반으로 하는 모바일 웹앱입니다.

중요한 전제:

- 이 앱은 실시간 관제 앱이 아닙니다.
- 실시간 콜 진행 상태를 보여주지 않습니다.
- 현재 위치나 GPS 이동 경로를 추적하지 않습니다.
- 관리자가 업로드한 최신 주차 엑셀 데이터를 기준으로 화면을 표시합니다.
- 베타 단계에서는 DB 저장 없이 브라우저에서 업로드한 데이터를 앱 상태로 반영합니다.
- 새로고침 후에는 다시 업로드가 필요할 수 있습니다.
- 정산앱과는 링크로 연결되며, 정산 계산 기능은 코칭센터에 포함하지 않습니다.

## 3. 접속 주소

라이더 코칭센터 Ver.2 베타 주소:

```text
https://rider-coaching-center-v2.vercel.app
```

정산 관리 앱 주소:

```text
https://ridersettle-fmyozrbw.manus.space/
```

## 4. 베타 테스트 계정

관리자 계정:

```text
아이디: admin
비밀번호: admin1234
```

라이더 계정:

```text
아이디: rider1
비밀번호: rider1234
```

승인대기 계정:

```text
아이디: pending
비밀번호: pending1234
```

운영 전에는 테스트 계정 안내를 숨기고, 승인된 개인 계정 방식으로 전환하는 것을 권장합니다.

## 5. 관리자모드 전체 흐름

관리자모드는 주차별 엑셀 데이터를 업로드하고, 검수한 뒤, 라이더별 코칭 화면에 반영하는 운영자용 화면입니다.

관리자 기본 흐름:

1. 관리자 계정으로 로그인합니다.
2. `업로드` 화면에서 쿠팡플러스 원천 엑셀을 선택합니다.
3. 파싱 미리보기와 검수 이슈를 확인합니다.
4. 문제가 없으면 `이번 주차 데이터로 반영`을 누릅니다.
5. `대시보드`에서 전체 운영 요약을 확인합니다.
6. `검수`에서 누락값과 확인 필요 항목을 확인합니다.
7. `코칭`에서 라이더별 코칭 메시지를 확인/수정합니다.
8. `더보기`에서 운영 로그, 페이스 체크 설정, 정산앱 링크를 확인합니다.

## 6. 관리자모드 기능별 설명

### 6.1 로그인

목적:

관리자 전용 화면에 접근하기 위한 시작 화면입니다.

사용 방법:

1. 베타 주소에 접속합니다.
2. 관리자 계정 `admin / admin1234`를 입력합니다.
3. 로그인 버튼을 누릅니다.

정상 동작:

- 로그인 후 관리자 화면으로 이동합니다.
- 하단 메뉴에 `대시보드`, `업로드`, `검수`, `코칭`, `더보기`가 보입니다.

주의사항:

- 관리자 계정은 업로드와 코칭 메시지 설정 권한이 있으므로 외부 라이더에게 공유하지 않습니다.
- 운영 전에는 공용 관리자 계정을 폐기하고 개인 관리자 계정으로 전환하는 것이 좋습니다.

### 6.2 대시보드

목적:

최신 반영 주차 기준으로 전체 운영 현황을 한눈에 확인하는 화면입니다.

주요 표시 항목:

- 완료 콜 수
- 운영 라이더 수
- 검수 이슈 수
- 노출 메시지 수
- 완료 콜 상위 라이더
- 최근 업로드 요약
- 마지막 반영 파일
- 마지막 반영 주차
- 마지막 반영 시간

관리자가 확인할 것:

- 업로드 전에는 샘플 데이터 표시 상태인지 확인합니다.
- 업로드 후 `이번 주차 데이터로 반영`을 누르면 대시보드 수치가 업로드 데이터 기준으로 바뀌는지 확인합니다.
- 완료 콜 수와 라이더 수가 업로드한 엑셀의 규모와 크게 다르지 않은지 확인합니다.
- 검수 이슈 수가 지나치게 많으면 `검수` 화면에서 원인을 확인합니다.

주의사항:

- 파일을 선택만 하면 대시보드 수치가 바뀌지 않습니다.
- 반드시 `이번 주차 데이터로 반영`을 눌러야 전체 화면 기준 데이터가 변경됩니다.

### 6.3 업로드

목적:

쿠팡플러스 원천 엑셀 파일을 브라우저에서 읽어 미리보고, 정상 데이터만 이번 주차 기준으로 반영하는 화면입니다.

허용 파일명 예시:

```text
동구바로_대전_동구중앙_2026_05-4.xlsx
```

차단 파일:

- 파일명에 `정산`이 포함된 파일
- 파일명에 `최종`이 포함된 파일
- 정산 최종 파일 계열
- 허용 패턴과 다른 이름의 파일

필수 시트:

```text
오더별 상세 내역서
```

업로드 처리 기준:

- 7행을 헤더로 사용합니다.
- B:Y 컬럼 범위를 기준으로 파싱합니다.
- 빈 행은 유효 오더 수에서 제외합니다.
- `0`은 `단건배달`로 처리합니다.
- `멀티배달1~멀티배달5`는 멀티배달 계열로 처리합니다.
- 파일명에서 `week_code`와 `week_label`을 추출합니다.

사용 방법:

1. `업로드` 화면으로 이동합니다.
2. 파일 선택 버튼을 누릅니다.
3. 쿠팡플러스 원천 엑셀 파일을 선택합니다.
4. 파일명 검수, 시트 존재 여부, 파싱 결과를 확인합니다.
5. 전체 행 수, 유효 오더 수, 검수 이슈를 확인합니다.
6. 라이더별 집계 미리보기를 확인합니다.
7. 문제가 없으면 `이번 주차 데이터로 반영`을 누릅니다.

버튼 설명:

- `이번 주차 데이터로 반영`: 현재 미리보기 데이터를 관리자/라이더 전체 화면 기준 데이터로 적용합니다.
- `미리보기 취소`: 미리보기만 삭제하고 기존 반영 데이터는 유지합니다.

주의사항:

- 업로드 파일 원본은 서버에 저장하지 않습니다.
- 브라우저 File 객체도 저장하지 않습니다.
- 잘못된 파일을 선택해도 기존 정상 반영 데이터는 유지됩니다.
- 새로고침 후에는 다시 업로드가 필요할 수 있습니다.

### 6.4 검수

목적:

업로드 데이터에서 분석에 필요한 값이 누락되었거나 확인이 필요한 항목을 점검하는 화면입니다.

상단 숫자 라벨:

- `총행수`: 엑셀에서 읽은 전체 데이터 행 수입니다.
- `유효 오더 후보`: 빈 행을 제외하고 오더로 볼 수 있는 행 수입니다.
- `확인 필요 행`: 하나 이상의 검수 이슈가 있는 행 수입니다.
- `이슈 건수`: 누락값이나 확인 필요 항목을 모두 더한 수입니다.

주의:

한 행에 여러 누락값이 있으면 이슈 건수는 중복 포함됩니다. 예를 들어 한 오더에 배달타입, 피크타임, 시간값이 모두 비어 있으면 이슈 건수는 3건으로 계산될 수 있습니다.

이슈 유형 설명:

- `배달타입 누락`: 단건/멀티 구분값이 비어 있습니다.
- `배달타입 확인`: `0`, `멀티배달1~5` 등 해석 가능한 값 외의 값이 있습니다.
- `피크타임 누락`: Post_Lunch, Post_Dinner 등 구간 분석값이 비어 있습니다.
- `이름 누락`: 라이더별 집계와 권한 매핑에 필요한 이름값이 비어 있습니다.
- `주문번호 누락`: 오더 중복 확인과 이력 표시 기준값이 비어 있습니다.
- `시간값 누락`: 배정/수락/배달/소요시간 등 시간 분석에 필요한 값이 비어 있습니다.

관리자가 확인할 것:

- 배달타입 누락이 많으면 멀티비율 분석이 부정확할 수 있습니다.
- 피크타임 누락이 많으면 점심/저녁 피크 참여율 분석이 제한됩니다.
- 시간값 누락이 많으면 출근횟수, 활동 패턴, 페이스 계산의 정확도가 낮아질 수 있습니다.
- 이름 또는 주문번호 누락은 라이더별 데이터 매핑에 직접 영향을 줄 수 있습니다.

현재 베타에서의 의미:

검수 화면은 데이터를 고치는 화면이 아니라, 원천 엑셀 데이터의 상태를 확인하는 화면입니다. 운영자는 이슈가 많은 경우 원천 파일이 맞는지, 시트/컬럼 구조가 맞는지 확인해야 합니다.

### 6.5 코칭

목적:

라이더별 자동 코칭 메시지를 확인하고, 관리자가 직접 수정하거나 노출 여부를 관리하는 화면입니다.

주요 기능:

- 라이더 선택
- 자동 메시지 확인
- 관리자 수정 메시지 입력
- 라이더 노출 ON/OFF 설정
- 내부 메모 작성
- 라이더 화면 미리보기 확인

기능별 설명:

- `라이더 선택`: 코칭 메시지를 확인할 라이더를 선택합니다.
- `자동 메시지`: 업로드 데이터와 집계 결과를 바탕으로 만들어진 기본 코칭 문구입니다.
- `관리자 수정 메시지`: 실제 라이더에게 보여줄 문구를 관리자가 조정하는 영역입니다.
- `노출 ON/OFF`: 라이더 화면에 코칭 메시지를 보여줄지 결정합니다.
- `내부 메모`: 관리자만 보는 기록입니다. 라이더에게 노출되지 않습니다.
- `라이더 화면 미리보기`: 라이더에게 실제로 어떻게 보일지 미리 확인합니다.

관리자가 확인할 것:

- 자동 메시지가 실제 운영 판단과 맞는지 확인합니다.
- 민감하거나 오해될 수 있는 표현은 관리자 수정 메시지에서 조정합니다.
- 노출 OFF 상태의 메시지는 라이더 화면에 보이지 않는지 확인합니다.
- 내부 메모에 라이더에게 보여서는 안 되는 운영 메모를 기록할 수 있습니다.

주의사항:

- 코칭 메시지는 라이더의 동기부여와 행동 조정에 영향을 줄 수 있으므로 단정적 표현보다 실행 가능한 짧은 문장으로 작성하는 것이 좋습니다.
- 베타 단계에서는 메시지 저장이 앱 상태 기준이므로 새로고침/재배포/세션 상태에 따라 초기화될 수 있습니다.

### 6.6 더보기

목적:

운영 보조 설정, 베타 안내, 로그, 외부 링크를 관리하는 화면입니다.

주요 기능:

- 운영 로그
- 페이스 체크 설정
- 베타 안내
- 베타 체크리스트
- 베타 제외 항목
- 정산 관리 앱 이동 링크

운영 로그:

업로드 미리보기 생성, 데이터 반영, 미리보기 취소, 정산/최종 파일 차단, 시트 누락, 파싱 실패, 페이스 체크 설정 변경 등의 기록을 최신순으로 보여줍니다.

페이스 체크 설정:

라이더 화면의 `오늘의 페이스 체크`에 적용되는 기본 문구와 기준값을 조정합니다.

설정 가능 항목:

- 기본 주간 목표
- 수면 경고 기준
- 위험 컨디션 문구
- 식사 건너뜀 문구
- 주간형 루틴 문구
- 야간형 루틴 문구
- 음악 모드 안전 문구

정산 관리 앱 링크:

정산앱으로 이동하는 링크입니다. 새 탭으로 열립니다.

주의사항:

- 페이스 체크 설정은 베타 단계에서 앱 내부 상태로만 반영됩니다.
- 새로고침 후 초기화될 수 있습니다.
- 정산앱은 별도 시스템이며, 코칭센터에서 정산 계산을 수행하지 않습니다.

## 7. 라이더모드 전체 흐름

라이더모드는 본인의 최신 업로드 주차 운행 데이터를 확인하고, 코칭 메시지와 오늘의 페이스를 점검하는 화면입니다.

라이더 기본 흐름:

1. 라이더 계정으로 로그인합니다.
2. `내현황`에서 이번 기준 주차의 요약을 확인합니다.
3. `오늘의 페이스 체크`에 오늘 상태를 입력합니다.
4. `내오더`에서 최신 업로드 주차의 본인 오더 이력을 확인합니다.
5. `내지도`에서 픽업/배달 지역 기반 미리보기를 확인합니다.
6. `내코칭`에서 코칭 포인트와 실천 미션을 확인합니다.
7. `MY`에서 베타 안내와 정산앱 링크를 확인합니다.

## 8. 라이더모드 기능별 설명

### 8.1 로그인

목적:

라이더 본인 화면에 접근하기 위한 시작 화면입니다.

사용 방법:

1. 베타 주소에 접속합니다.
2. 라이더 계정 `rider1 / rider1234`를 입력합니다.
3. 로그인 버튼을 누릅니다.

정상 동작:

- 로그인 후 라이더 전용 화면으로 이동합니다.
- 하단 메뉴에 `내현황`, `내오더`, `내지도`, `내코칭`, `MY`가 보입니다.
- 다른 라이더를 선택하는 UI는 보이지 않아야 합니다.
- 업로드, 검수, 관리자 코칭 관리 화면은 보이지 않아야 합니다.

### 8.2 내현황

목적:

최신 업로드 주차 기준으로 내 운행 요약과 오늘의 페이스 체크를 확인하는 화면입니다.

주요 표시 항목:

- 완료 콜 수
- 출근횟수
- 멀티비율
- 피크타임 참여율
- 오늘의 페이스 체크
- 주간 운행 요약

라이더가 확인할 것:

- 표시된 주차가 최신 업로드 주차인지 확인합니다.
- 완료 콜 수와 본인이 기억하는 지난주 운행 규모가 크게 다르지 않은지 확인합니다.
- 멀티비율과 피크타임 참여율을 보고 본인의 운행 패턴을 이해합니다.

주의사항:

- 내현황은 실시간 오늘 운행 현황이 아닙니다.
- 관리자가 업로드하고 반영한 최신 주차 데이터 기준입니다.

### 8.3 오늘의 페이스 체크

목적:

지난주 확정 데이터와 라이더의 수기 입력을 바탕으로 현재 주차 목표와 컨디션을 점검하는 화면입니다.

입력 항목:

- 오늘 완료 콜 수
- 오늘 운행 시작 시간
- 현재 컨디션: 좋음 / 보통 / 피곤 / 위험
- 식사 여부: 완료 / 아직 / 건너뜀
- 수면 시간
- 휴식 여부: 충분 / 부족 / 없음
- 이번 주 목표 콜 수
- 루틴 유형: 주간형 / 야간형
- 음악 모드: 콜 수행 템포 / 휴식 릴랙스 / 수면 유도

표시 항목:

- 지난주 기준 페이스
- 목표까지 필요한 추가 콜 수
- 현재 입력 기준 권장 페이스
- 다음 추천 행동
- 주간형/야간형 루틴 코칭
- 음악 모드 안내

추천 행동 예시:

- 오늘 페이스가 낮으면 피크타임 집중형 운행을 권장합니다.
- 수면 시간이 부족하면 짧은 집중 운행과 휴식을 권장합니다.
- 컨디션이 위험이면 목표보다 안전을 우선하라는 문구가 표시됩니다.
- 식사를 건너뛰면 피크타임 전 식사나 수분 보충을 권장합니다.

주의사항:

- 입력값은 이 기기 기준 임시 저장입니다.
- 서버 저장이나 DB 저장은 베타 단계에서 제공하지 않습니다.
- 음악 모드는 추천 UI만 있으며 실제 음악 재생은 없습니다.
- 운행 중 조작을 유도하지 않습니다.

### 8.4 내오더

목적:

최신 업로드 주차의 본인 오더 이력을 확인하는 화면입니다.

표시 항목:

- 주문번호
- 배달타입
- 피크타임
- 픽업 지역
- 배달 지역
- 시간 정보
- 거리
- 정산금액

라이더가 확인할 것:

- 본인 오더만 표시되는지 확인합니다.
- 다른 라이더의 오더가 보이지 않는지 확인합니다.
- 배달타입과 피크타임 정보가 실제 운행 패턴과 맞는지 확인합니다.

주의사항:

- 내오더는 실시간 오더 목록이 아닙니다.
- 현재 배달 중인 콜이나 진행 중 상태를 보여주지 않습니다.
- 최신 업로드 주차의 과거 운행 이력입니다.

### 8.5 내지도

목적:

최신 업로드 주차의 픽업/배달 지역 분포를 확인하는 화면입니다.

표시 내용:

- 픽업 지역 기반 미리보기
- 배달 지역 기반 미리보기
- 주요 활동 지역을 이해하기 위한 참고 정보

라이더가 확인할 것:

- 본인의 주요 픽업/배달 지역 흐름을 이해합니다.
- 특정 지역에 운행이 몰려 있는지 확인합니다.

주의사항:

- 내지도는 현재 위치 지도가 아닙니다.
- GPS 추적, 실시간 이동 경로, 현재 배달 경로를 제공하지 않습니다.
- 위치 기반 관제 기능이 아닙니다.

### 8.6 내코칭

목적:

최신 업로드 주차 운행 데이터를 바탕으로 라이더별 코칭 포인트와 실천 미션을 확인하는 화면입니다.

표시 항목:

- 배차 친화 점수
- 구간별 참여율
- 출근/활동 패턴
- 배달타입/멀티 분석
- 코칭 포인트
- 실천 미션
- 관리자 노출 메시지

라이더가 확인할 것:

- 본인의 강점과 개선 포인트를 확인합니다.
- 점심/저녁 피크 참여율을 확인합니다.
- 멀티비율과 활동 패턴을 확인합니다.
- 실천 미션을 다음 주 운행 계획에 반영합니다.

주의사항:

- 코칭은 참고용이며, 실제 운행 판단은 안전과 컨디션을 우선합니다.
- 관리자가 노출 OFF로 설정한 메시지는 보이지 않을 수 있습니다.

### 8.7 MY

목적:

라이더 계정 정보, 베타 안내, 정산앱 이동 링크를 확인하는 화면입니다.

주요 기능:

- 베타 안내 확인
- 정산앱으로 이동
- 현재 앱이 실시간 관제 앱이 아니라는 안내 확인

정산앱 링크:

정산앱은 새 탭으로 열립니다.

```text
https://ridersettle-fmyozrbw.manus.space/
```

주의사항:

- 정산앱은 별도 시스템입니다.
- 코칭센터에서 정산 계산이나 정산 엑셀 파싱을 수행하지 않습니다.

## 9. 승인대기 계정 설명

승인대기 계정은 아직 사용 승인되지 않은 사용자의 화면을 확인하기 위한 테스트 계정입니다.

정상 동작:

- 관리자 화면에 접근할 수 없습니다.
- 라이더 데이터에 접근할 수 없습니다.
- 승인 대기 안내 화면이 표시됩니다.

운영 전 정책:

- 실제 운영에서는 관리자가 라이더 계정을 승인한 뒤에만 라이더 화면 접근을 허용하는 방식이 필요합니다.

## 10. 이슈 유형 상세 설명

검수 화면의 이슈 유형은 업로드 데이터가 코칭 분석에 충분한지 확인하기 위한 항목입니다.

### 10.1 배달타입 누락

뜻:

단건배달/멀티배달 구분값이 비어 있는 상태입니다.

영향:

- 단건/멀티 분석이 제한됩니다.
- 멀티비율 계산이 부정확해질 수 있습니다.
- 배달타입 기반 코칭 문구의 정확도가 낮아질 수 있습니다.

### 10.2 배달타입 확인

뜻:

해석 가능한 값 외의 배달타입 값이 들어온 상태입니다.

정상 처리 예시:

- `0`은 단건배달로 처리합니다.
- `멀티배달1~멀티배달5`는 멀티배달 계열로 처리합니다.

영향:

- 예상하지 못한 값이 있으면 운영자가 원천 파일 구조를 확인해야 합니다.

### 10.3 피크타임 누락

뜻:

Post_Lunch, Post_Dinner 등 구간 분석에 필요한 피크타임 값이 비어 있는 상태입니다.

영향:

- 점심/저녁 피크 참여율 계산이 제한됩니다.
- 구간별 코칭 정확도가 낮아집니다.
- 피크타임 집중형 운행 조언이 약해질 수 있습니다.

### 10.4 이름 누락

뜻:

라이더 이름값이 비어 있는 상태입니다.

영향:

- 라이더별 집계가 제한됩니다.
- 권한 매핑이나 화면 표시가 부정확할 수 있습니다.

### 10.5 주문번호 누락

뜻:

오더 기준값인 주문번호가 비어 있는 상태입니다.

영향:

- 오더 중복 확인이 어렵습니다.
- 내오더 이력 표시 기준이 약해집니다.

### 10.6 시간값 누락

뜻:

배정, 수락, 배달, 소요시간 등 시간 분석에 필요한 값이 비어 있는 상태입니다.

영향:

- 출근횟수 판단이 제한됩니다.
- 활동 패턴 분석이 약해집니다.
- 시간당 페이스 계산이 제한될 수 있습니다.

- 본인 데이터만 표시

## 13. 문제 발생 시 확인할 것

엑셀 업로드가 안 될 때:

- 파일명에 `정산` 또는 `최종`이 포함되어 있는지 확인합니다.
- 기본 시트명이 `오더별 상세 내역서`인지 확인합니다.
- 파일이 `.xlsx` 형식인지 확인합니다.
- 허용 파일명 패턴과 맞는지 확인합니다.

수치가 이상할 때:

- 파일 선택 후 `이번 주차 데이터로 반영`을 눌렀는지 확인합니다.
- 새로고침 후라면 다시 업로드가 필요한지 확인합니다.
- 검수 화면에서 누락 이슈가 많은지 확인합니다.
- 엑셀 원천 파일의 배달타입, 피크타임, 시간값 컬럼이 비어 있는지 확인합니다.

라이더 데이터가 안 보일 때:

- 로그인한 계정의 rider_id와 업로드 데이터가 매칭되는지 확인합니다.
- 최신 업로드 주차에 해당 라이더 오더가 있는지 확인합니다.
- 관리자 화면에서 업로드 데이터를 실제 반영했는지 확인합니다.

정산앱 링크가 안 열릴 때:

- 브라우저 팝업 차단 여부를 확인합니다.
- 정산앱 URL이 올바른지 확인합니다.
- 관리자 더보기 또는 라이더 MY의 링크 노출 여부를 확인합니다.

Vercel 주소가 안 열릴 때:

- 주소가 `https://rider-coaching-center-v2.vercel.app`인지 확인합니다.
- 브라우저 새로고침을 시도합니다.
- 배포 상태가 Vercel에서 Ready인지 확인합니다.
